## Commands

```bash
# Development (Vite on 5173, Express on 3001)
npm run dev

# Production build + start (serves both on 3000)
npm run build
npm run start

# Type-checking
npx tsc --noEmit --project tsconfig.server.json   # server
npx tsc --noEmit --project tsconfig.json           # client

# Database
npm run db:push       # apply schema changes
npm run db:generate   # regenerate Prisma client

# Testing
npm test              # run all tests (vitest)
npm run test:watch    # watch mode
```

Tests use Vitest + Supertest. Test files are in `src/server/__tests__/`. Auth is mocked (passthrough with fixed test user). A separate `prisma/test.db` is used. No linting configured.

## Architecture

**Express 5 backend** (`src/server/`) serves a **React 19 SPA** (`src/client/`) with **SQLite via Prisma** (`prisma/schema.prisma`). In dev, Vite proxies `/api` to Express. In production, Express serves the built SPA and API on one port.

### Server

- **Entry**: `src/server/server.ts` — Express setup, static serving, startup migrations
- **Routes**: `src/server/routes.ts` — all API endpoints (large file, ~1600 lines)
- **Auth**: `src/server/auth.ts` (better-auth config) + `src/server/authMiddleware.ts` (`requireAuth` middleware attaches `userId`/`userEmail`/`userName` to `req`)
- **Database**: `src/server/prisma.ts` — singleton Prisma client with better-sqlite3 adapter
- **AI**: `src/server/ai.ts` (OpenRouter) + `src/server/chat.ts` (TanStack AI chat with tool-calling, SSE streaming)
- **Import**: `src/server/importManager.ts` — background job queue for file import processing

### Client

- **Entry**: `src/client/main.tsx`
- **Routing**: TanStack Router with file-based routes in `src/client/routes/`. Route tree auto-generated by Vite plugin (`routeTree.gen.ts` — do not edit).
- **Data fetching**: TanStack Query hooks in `src/client/api/hooks.ts`, fetch helpers in `src/client/api/client.ts`
- **Auth client**: `src/client/lib/auth.ts` — `useSession()`, `signOut`, `authClient`
- **All fetch calls** use `credentials: 'include'`; 401 responses redirect to `/login`

### Parser System (`src/server/parsers/`)

Modular parsers for bank statement import:
- `registry.ts` — auto-detects format (CSV by header, MT940 by `:20:` tag, CAMT.052 by XML namespace)
- `templateParser.ts` — generic CSV parser driven by column-mapping config
- `mt940Parser.ts` — SWIFT MT940 format (`.mta`/`.sta` files)
- `camt052Parser.ts` — ISO 20022 XML (`.xml` files)
- `builtinTemplates.ts` — built-in configs for C24, Volksbank, OLB
- `csvUtils.ts` — shared utilities (encoding detection, delimiter detection, date/amount parsing)

## Key Conventions

### Express 5 Specifics
- Wildcard routes use `{*splat}` syntax, not `*`
- `req.params.id` returns `string | string[]` — always cast with `as string`

### Prisma / Database
- Column names are camelCase in DB (`userId`, not `user_id`) — Prisma default
- For models with composite `@@unique` (Setting, Category): use `findFirst` with filters, not `findUnique`
- Raw SQL for transactions must join through bank_accounts: `INNER JOIN bank_accounts a ON t.account_id = a.id AND a.userId = ?`
- BigInt from raw queries needs serialization to Number for JSON responses
- In raw SQL, use single quotes for `strftime` format strings: `strftime('%Y', ...)` not `strftime("%Y", ...)` — better-sqlite3 treats double quotes as identifiers

### Multi-User Data Scoping
- `userId` column on: `bank_accounts`, `categories`, `category_rules`, `bank_templates`, `settings`, `import_log`
- Transactions scoped indirectly via `account_id → bank_accounts.userId`
- Built-in templates (`is_builtin: true`) are globally accessible regardless of userId
